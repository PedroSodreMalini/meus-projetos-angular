# Go-Task -> Organizando em arquitetura do Angular 20

## Resumo
+ Kanbam de tarefas com Angular, Tailwind e persistência local.
+ Figma: [Link](https://www.figma.com/design/rX2D4XX2V5Wq3QtTQOCUh1/GoTask--Community-?node-id=0-1&p=f&t=1jTH1GsqbMbxrN4H-0)
+ Diferente do projeto 1, este segue a nova arquitetura Core, Features e Shared do Angular 20.

## Conceitos utilizados
1. Arquitetura Angular 20 (Core, Shared, Features).
2. Componentização.
3. Separação de responsabilidades entre services e componentes.
4. Desacoplamento entre componentes.
5. Padrão de projeto.
6. Fluxo de dados entre componentes.
7. Gerenciamento de estado(fonte de verdade).
8. Imutabilidade de dados.
9. Padronização de nomes e organização de arquivos.
10. Mobile First e Tailwind.
11. Angular Material CDK(Dialog e Drag and Drop).
12. RXJS(Behaviour Subject).
13. Pipes (AsyncPipe, SlicePipe).
14. @let, @if, @for.
15. Decorators(@Input).
16. Injeção de dependências (Services).
17. Interpolação, Property Binding e Event Binding.

## Flxuo do projeto
1. Criação do projeto inicial.
2. Criação do layout dos componentes.
3. Criação do fluxo de dados.
4. Criação da lógica dos componentes.
5. Persistindo as tarefas em LocalStorage.
   
## Como rodar
```bash
npm i
npm run dev
```
